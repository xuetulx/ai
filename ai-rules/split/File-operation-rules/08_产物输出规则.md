## 9. 产物输出规则

### 9.1 产物存放规范（v2.0 统一）

1. `[DO]` 最终产物**优先**放入 `output/` 目录（项目默认输出目录）
2. `[DO]` 亦可放入 `dist/`、`build/`、`result/` 目录（适用于构建/结果类产物）
3. `[DO]` 产物文件命名清晰，包含任务标识和版本号
   - 示例：`radar_analysis_report_v1.0.md`
   - 示例：`data_process_script_v2.1.py`
4. `[DO]` 多文件产物建议创建子目录分类存放
   - 示例：`output/report_20260701/` 内含报告、图表、数据等

### 9.2 产物交付规范

1. `[DO]` 产物生成后在回复中明确告知文件路径和文件名
2. `[DO]` 重要产物附带简要说明：内容、用途、使用方法
3. `[DO]` 多产物时列清单：文件名 + 简要说明
4. `[DO]` 确保产物文件完整、可正常打开/运行
