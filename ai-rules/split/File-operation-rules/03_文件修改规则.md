## 4. 文件修改规则

### 4.1 受保护文件清单（修改前必须 `[ASK]`）

以下文件属于项目核心资产，修改必须走副本确认流程：

| 类别 | 文件/目录 |
|------|----------|
| **AI 指令/规则文件** | `CLAUDE.md`、`AGENTS.md`、`AI_RULES_*.md`、`*.rules`、`.cursorrules`、`.cursor/rules/` |
| **项目元文件** | `README.md`、`LICENSE`、`CHANGELOG.md`、`VERSION_LOG.md` |
| **构建/依赖配置** | `package.json`、`Cargo.toml`、`go.mod`、`pyproject.toml`、`requirements.txt` |
| **依赖锁定文件** | `package-lock.json`、`yarn.lock`、`Cargo.lock`、`go.sum`、`pnpm-lock.yaml` |
| **CI/CD 配置** | `.github/workflows/`、`.gitlab-ci.yml`、`Jenkinsfile`、`Dockerfile` |
| **环境/密钥** | `.env`、`.env.*`、`*.pem`、`*.key`、`*.cert`、`config*.yaml` |
| **Git 配置** | `.gitignore`、`.gitattributes`、`.git/config` |
| **编辑器配置** | `.vscode/`、`.idea/`、`.editorconfig` |

### 4.2 修改流程

#### 4.2.1 用户原始文件修改流程

```
[DO]  1. 读取原文件，理解内容
[DO]  2. 创建备份副本：原文件名.bak 或 原文件名_backup_日期
[DO]  3. 在副本或新文件上进行修改
[ASK] 4. 向用户展示修改差异和修改理由，确认是否应用
[DO]  5. 用户确认后覆盖原文件（保留备份或根据用户指示删除）
```

#### 4.2.2 AI 生成文件修改流程

```
[DO] 1. 读取 AI 之前生成的文件
[DO] 2. 直接修改或创建新版本（v2.0）
[DO] 3. 修改后在回复中说明变更点
```

### 4.3 修改约束

1. `[DO]` 修改已有代码时优先保持原有代码风格和缩进格式，不随意重格式化整个文件
2. `[DO]` 如果新增代码片段，遵循 AI_RULES_编程参考规范（4 空格缩进、命名规范等）
3. `[DO]` 增量修改优先，保留原有逻辑，不做无关的重构
4. `[DO]` 修改后保留文件原有的注释和文档
5. `[BLOCK]` 不得删除用户写的注释或标记为 TODO/FIXME 的内容
