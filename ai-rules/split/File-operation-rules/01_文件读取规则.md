## 2. 文件读取规则

### 2.1 读取权限（统一细分表）

| 文件类型 | 读取权限 | 注意事项 |
|---------|---------|---------|
| 源代码文件 | `[DO]` 可自由读取 | 不得将敏感代码泄露到外部 |
| 配置文件（不含密钥） | `[DO]` 可读取 | 敏感值用 `[REDACTED]` 替代后输出 |
| 文档文件 | `[DO]` 可读取 | 尊重文档版权和隐私 |
| 依赖目录文件 | `[DO]` 可读取 | 不修改依赖包内的文件 |
| `.git/` 内部 | `[BLOCK]` 禁止直接读取 | 使用 git 命令间接获取信息 |
| 环境变量文件 (`.env`/`.env.*`) | `[ASK]` 需确认用途 | 读取后不得输出明文内容，敏感值须脱敏 |
| 凭证/密钥/证书 (`*.pem`/`*.key`/`*.cert`/`credentials.*`/`token*`) | `[BLOCK]` 禁止读取和外传 | 绝对禁止，不可通过 ASK 绕过 |
| 包含密码的配置文件 (`config*.yaml`/`settings.py` 等) | `[ASK]` 读取前确认 | 读取时脱敏输出，密码字段用 `***` 替代 |

### 2.2 读取约束

1. `[DO]` 读取大文件（> 10MB）时优先使用分页/分段读取，避免一次性加载全部内容
2. `[DO]` 二进制文件（图片、音频、压缩包等）只在必要时读取，且不输出原始内容
3. `[DO]` 读取文件时保持只读模式，不得意外修改文件时间戳或属性
4. `[ASK]` 需要批量读取 ≥ 20 个文件时，先告知用户范围和目的
