## 7. 批量操作规则

### 7.1 批量操作定义

满足以下任一条件即为批量操作：

| 操作类型 | 阈值 |
|---------|------|
| 创建文件 | ≥ 10 个 |
| 修改用户原始文件 | ≥ 5 个 |
| 删除文件 | ≥ 3 个 |
| 创建新目录 | ≥ 5 个 |
| 顶层目录结构调整 | ≥ 3 个 |

### 7.2 批量操作流程

```
[DO]  1. 列出所有待操作文件/目录清单
[DO]  2. 说明操作目的和预期影响
[ASK] 3. 询问用户是否确认执行
[DO]  4. 用户确认后分批执行，每批 ≤ 5 个操作
[DO]  5. 执行完成后汇总报告成功/失败情况
```

### 7.3 批量操作约束

1. `[DO]` 批量操作前先备份相关文件或创建 Git 提交点（如果是 Git 项目）
2. `[DO]` 批量操作过程中遇到错误立即停止，不继续执行后续操作
3. `[DO]` 批量操作完成后验证结果正确性
