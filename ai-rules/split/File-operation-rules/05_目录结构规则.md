## 6. 目录结构规则

### 6.1 标准项目目录识别

Agent AI 应能识别并遵循以下常见项目结构：

```
项目根目录/
├── src/           # 源代码
├── lib/           # 库文件
├── tests/         # 测试代码
├── docs/          # 文档
├── config/        # 配置文件
├── scripts/       # 脚本工具
├── data/          # 数据文件
├── assets/        # 静态资源
├── output/        # 输出产物（AI 可写，首选）
├── dist/          # 构建产物（AI 可写）
├── build/         # 构建产物（AI 可写）
├── result/        # 结果产物（AI 可写）
├── tmp/           # 临时文件（AI 可写）
├── .temp/         # 临时文件（AI 可写）
├── .cache/        # 缓存文件（AI 可写）
├── .claude/       # Claude Code 指令/配置/S skills/H hooks（只读，不手动修改）
├── .cursor/       # Cursor 指令/配置（只读，不手动修改）
├── node_modules/  # 依赖（只读，不手动修改）
├── .venv/         # 虚拟环境（只读）
└── .git/          # Git 目录（禁止直接操作）
```

### 6.2 目录操作约束

1. `[DO]` 优先使用项目已有的目录结构，不随意创建新的顶层目录
2. `[ASK]` 需要新增顶层目录时，说明用途并询问用户
3. `[DO]` 创建子目录时遵循已有命名风格（全小写 + 下划线/连字符）
4. `[BLOCK]` 不得重命名或移动项目已有的核心目录
